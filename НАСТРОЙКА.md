## 2. ПРОБЛЕМА С EMAIL - НЕ ПРИХОДЯТ УВЕДОМЛЕНИЯ

### Возможные причины:

#### ❌ 1. Неправильный пароль приложения Mail.ru

**Решение:**
1. Зайдите на https://mail.ru
2. Настройки (шестерёнка) → **Безопасность**
3. Включите **двухфакторную аутентификацию** (если не включена)
4. Прокрутите до **"Пароли для внешних приложений"**
5. Создайте **новый пароль** для "Почта"
6. Скопируйте пароль (он показывается только 1 раз!)
7. Вставьте в `.env`:
   ```env
   MAIL_PASSWORD=ваш_новый_пароль_приложения
   ```

#### ❌ 2. Неправельные настройки SMTP

**Для Mail.ru используйте:**
```env
MAIL_SERVER=smtp.mail.ru
MAIL_PORT=465
MAIL_USE_TLS=False
MAIL_USE_SSL=True
```

#### ❌ 3. Блокировка фаерволом

**Проверьте на сервере:**
```bash
# Открыть порт 465
ufw allow 465/tcp

# Или
iptables -A OUTPUT -p tcp --dport 465 -j ACCEPT
```

## 3. ПРОВЕРКА EMAIL

### Локальный тест на сервере:
```bash
cd /var/www/lift_site
source venv/bin/activate
python3 << EOF
from flask_mail import Message
from app import app, mail

with app.app_context():
    msg = Message(
        "Тестовое письмо",
        recipients=["daniillomeyko@mail.ru"],
        body="Это тестовое письмо из системы ООО ЭТАЖИ"
    )
    try:
        mail.send(msg)
        print("✅ Письмо отправлено!")
    except Exception as e:
        print(f"❌ Ошибка: {e}")
EOF
```

### Просмотр логов:
```bash
tail -50 /var/www/lift_site/etazhi.log
```

Ищите строки:
```
INFO - Email уведомление отправлено администратору
WARNING - Не удалось отправить email: ...
```

## 4. АЛЬТЕРНАТИВЫ, ЕСЛИ MAIL.RU НЕ РАБОТАЕТ

### Вариант A: Gmail (требуется App Password)
```env
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_USE_TLS=True
MAIL_USE_SSL=False
MAIL_USERNAME=ваш_email@gmail.com
MAIL_PASSWORD=ваш_пароль_приложения_gmail
```

### Вариант B: Яндекс.Почта
```env
MAIL_SERVER=smtp.yandex.ru
MAIL_PORT=465
MAIL_USE_TLS=False
MAIL_USE_SSL=True
MAIL_USERNAME=ваш_email@yandex.ru
MAIL_PASSWORD=ваш_пароль_приложения_yandex
```

### Вариант C: Отключить email (только база данных)
В `app.py` найдите блок `try: if app.config['MAIL_USERNAME']:` и закомментируйте его.
